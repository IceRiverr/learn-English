# Feature 0006：逐句循环次数与自动下一句

状态：设计中

## 1. 结论

建议实现。

当前播放器的“当前句循环”是无限循环。它适合集中攻克一句，但不适合连续完成整课精听：
用户必须手动关闭循环或点击下一句，无法在走路、通勤或锁屏时按固定次数完成逐句训练。

本功能将循环扩展为有限次数逐句训练：

- 用户开启逐句循环后，当前句默认完整播放 3 次；
- 达到次数后自动进入下一句，并重新从第 1 次开始计数；
- 用户可以选择 1、2、3、5 次或无限循环；
- 最后一句完成后暂停，不自动回到课程开头。

“默认 3 次”只表示循环次数的默认偏好，不表示打开课程后默认开启循环。普通播放行为保持不变，
用户仍需主动点击“逐句循环”进入训练模式。

## 2. 要解决的问题

1. 无限循环没有自然终点，用户容易停留在一句而忘记继续。
2. 每听完一句都手动点击下一句，不适合免手持或锁屏训练。
3. 跟读通常需要固定重复节奏，当前界面无法表达“这是第几遍”。
4. 很短的句子无限重复时体验突兀，也容易造成误操作。

如果有限次数完成后只暂停，用户仍然需要逐句操作，不能真正解决连续训练问题。因此首版必须包含
“完成后自动下一句”，而不是只增加一个停止计数器。

## 3. 用户体验

控制区在现有“当前句”循环按钮旁增加次数控制：

```text
[↻ 逐句循环]  [−] [3 次] [+]  [中文 开]
```

循环开启后，按钮显示当前进度：

```text
[↻ 2 / 3]
```

无限循环显示：

```text
[↻ 4 / ∞]
```

次数按以下顺序切换：

```text
1 → 2 → 3 → 5 → ∞
```

- `−` 和 `+` 只在上述值之间移动，不允许输入任意数字。
- 循环关闭时仍可调整次数，以便先选择再开启。
- 次数按钮使用清晰的 `aria-label`，当前循环按钮继续使用 `aria-pressed`。
- 移动端保持同一行；必要时缩小按钮间距，不增加弹窗或设置页面。

## 4. 循环语义

### 4.1 启用循环

用户点击“逐句循环”时：

1. 将当前显示句设为循环目标；
2. 跳回该句 `start - 0.15` 秒；
3. 将当前次数设为 `1`；
4. 开始播放。

即使用户在一句中间开启循环，也从句首重新开始，避免把半句算作第一次。

### 4.2 有限次数

假设次数为 3：

```text
当前句第 1 次结束
→ 等待 300ms
→ 回到当前句，显示 2/3

当前句第 2 次结束
→ 等待 300ms
→ 回到当前句，显示 3/3

当前句第 3 次结束
→ 等待 300ms
→ 跳到下一句，显示 1/3
```

第一次播放计入次数，因此“3 次”表示总共播放三遍，不是首次播放后再额外播放三遍。

### 4.3 无限次数

无限模式保留当前行为：每次到达片段末尾后回到同一句。界面仍递增已完成次数，让用户知道自己
已经听了多少遍，但不自动进入下一句。

### 4.4 最后一句

最后一句达到有限次数后：

- 暂停在该句末尾；
- 保持显示 `3/3`；
- 不播放无字幕片尾，也不跳回课程第一句；
- 用户再次点击播放时，从最后一句句首开始新一轮 `1/3`。

普通播放未开启循环时，仍可继续播放最后字幕后的片尾音频。

## 5. 手动操作规则

### 5.1 上一句、下一句和字幕点击

循环开启时，用户点击上一句、下一句或任意字幕行：

- 新句成为循环目标；
- 计数重置为 `1`；
- 从新句句首开始播放。

循环关闭时保持当前行为。

### 5.2 时间轴和前后五秒

循环开启时，任何手动 seek 都重新确定所在片段并将次数重置为 `1`：

- seek 到另一句：循环目标切换到新句；
- seek 仍在当前句：目标不变，但本遍重新计为第 1 次。

统一重置比尝试推断“用户是否听完这一遍”更可靠，也更容易理解。

### 5.3 修改次数

- 增加次数：当前句继续播放，新的上限立即生效。
- 减少次数且当前计数已超过新上限：将显示计数收敛到新上限，当前遍结束后进入下一句。
- 切换到无限：保留当前已播放次数，之后不再自动下一句。
- 从无限切回有限：当前遍结束时根据新的上限决定重播或下一句。

### 5.4 关闭循环

关闭循环只退出逐句训练模式，不跳转、不暂停，也不改变当前播放位置。之后恢复普通连续播放。

切换课程或返回课程列表时继续沿用现有行为：关闭循环并清除本次计数。

## 6. 状态设计

在 `App.tsx` 中增加两个状态，不引入 Context、状态管理库或新分层：

```ts
type RepeatLimit = 1 | 2 | 3 | 5 | "infinite";

const [repeatLimit, setRepeatLimit] = useState<RepeatLimit>(3);
const [repeatIteration, setRepeatIteration] = useState(1);
```

现有 `loopSegment?: number` 继续表示循环是否开启以及目标片段。

为了避免音频事件读取过期的 React state，可以为循环目标、当前次数和切换锁增加对应 `ref`，但不把
它们抽成 hook 或 service。React state 用于渲染，ref 用于前后台音频回调中的即时判断。

## 7. 偏好与进度持久化

循环次数是全局界面偏好，使用一个新的 `localStorage` 键：

```text
segment-repeat-count
```

- 首次使用默认 `3`；
- 只允许 `1`、`2`、`3`、`5` 和 `infinite`；非法旧值回退到 `3`；
- 读取或写入失败时只使用 React state，不影响播放。

不保存以下状态：

- 当前是否开启循环；
- 当前正在循环哪一句；
- 当前已经播放到第几次。

这些是一次播放会话中的瞬时状态。`SavedProgress` 继续只保存 `currentTime` 和 `playbackRate`，不修改
IndexedDB 结构，也不需要数据库迁移。

## 8. 前台与后台播放

当前实现有两条片段结束检测路径：

- 页面可见时由 `requestAnimationFrame` 检测；
- 页面隐藏或锁屏时由 `<audio onTimeUpdate>` 检测。

加入计数后，两条路径必须调用同一个片段结束处理函数，例如：

```ts
completeRepeatIteration(audio)
```

该函数统一决定：

1. 重播当前句；
2. 自动进入下一句；
3. 无限循环继续计数；
4. 最后一句暂停。

使用一个切换锁避免 `requestAnimationFrame` 和 `timeupdate` 在同一片段末尾各计数一次。切换锁在完成
seek 并重新开始播放后释放。不能在前台和后台分支各自维护一套次数逻辑。

前台继续保留现有 300ms 间隔。后台浏览器可能限制定时器，因此以“不会重复计数、能够可靠回到目标
时间”为优先；若 300ms 等待在锁屏状态不可靠，可以像当前版本一样立即 seek。

## 10. 不实现

- 每句分别配置不同次数。
- 按课程保存不同循环次数。
- 自定义任意数字输入。
- A-B 自定义区间循环。
- 句间等待时长设置。
- 自动录音、跟读评分或语音识别。
- 完成整课后的训练统计。

这些功能可以在验证固定次数逐句训练有效后再单独设计。

## 11. 风险与处理

### 11.1 重复计数

前台检测和后台 `timeupdate` 可能同时观察到片段结束。必须使用统一处理函数和切换锁，保证一次越界只
增加一次计数。

### 11.2 很短的片段

短片段可能在 seek 后很快再次到达末尾。切换锁不能仅依赖下一次 `timeupdate` 释放，应在设置播放位置
并开始新一遍时明确释放。

### 11.3 浏览器后台节流

锁屏时 `requestAnimationFrame` 停止，`timeupdate` 频率也可能降低。允许越过末尾少量时间，但不能漏掉
自动下一句或把同一次结束计算两遍。

### 11.4 播放进度

循环过程中现有五秒节流保存可能记录句首附近的位置，这是合理的。刷新后恢复播放位置，但不恢复循环
模式和遍数，避免用户一打开课程就意外继续循环。

## 12. 自动与人工验证

优先把“到达片段末尾后的动作”写成纯状态转移并覆盖：

- 1/3 和 2/3 返回当前句；
- 3/3 进入下一句并重置到 1/3；
- 无限模式始终返回当前句并递增显示次数；
- 最后一句 3/3 返回暂停动作；
- 减少上限时计数正确收敛；
- 循环关闭时不进入次数逻辑。

浏览器人工验证：

1. 在一句中间开启循环，确认先回到句首并显示 1/3。
2. 完整听三遍后自动进入下一句。
3. 点击字幕、上一句、下一句和拖动时间轴后计数重置。
4. 在 1、2、3、5 和无限之间切换。
5. 最后一句完成后暂停，不回到第一句。
6. 锁屏或切到后台后仍准确完成三遍并进入下一句。
7. 关闭循环后恢复连续播放。
8. 刷新页面后次数偏好保留，但循环模式不自动开启。
9. 已下载课程无需重新下载音频，播放与进度保存不回归。

## 13. 验收标准

- 循环次数首次默认为 3，用户可以选择 1、2、3、5 或无限。
- 开启循环时从当前句句首开始，并显示当前遍数。
- 有限次数完成后自动进入下一句。
- 最后一句完成后暂停。
- 手动切句和 seek 会正确更新循环目标并重置计数。
- 前台、后台和锁屏状态下一次片段结束只计数一次。
- 次数偏好刷新后保留，循环开关和当前遍数不持久化。
- 普通播放、中文显示、播放速度、下载、离线播放和进度恢复不回归。
- 不新增后端、路由、状态管理库或 IndexedDB 迁移。
