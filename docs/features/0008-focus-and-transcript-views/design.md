# Feature 0008：当前句与全文双视图

状态：已实现，包含 iOS 主屏幕 PWA 锁屏循环修复

## 1. 结论

课程正文改为两个互斥视图：

- **当前句**：进入课程时默认显示，只展示正在播放的句子；
- **全文**：展示完整字幕，高亮并跟随当前播放句。

页面只通过“当前句 / 全文”按钮切换视图。切换只改变呈现，不暂停、不播放、不 seek，也不改变逐句循环或
保存进度。

## 2. 界面与行为

课程页保留三块稳定区域：

```text
[ 返回课程 ]
课程标题                         [删除本地课程]

[ 当前句 ] [ 全文 ]

┌──────────────────────────────┐
│ 当前选择的正文视图             │
└──────────────────────────────┘

[ 固定播放器底栏 ]
```

课程标题下方增加正文视图切换：

```text
[ 当前句 ] [ 全文 ]
```

使用两个普通按钮和 `aria-pressed` 表示选中状态，触控高度不小于 44px。

### 2.1 当前句

- 只渲染现有 `.focus-sentence`，不同时渲染全文列表。
- 继续显示句子序号、说话人、英文和可用翻译。
- 内容随 `currentSegment` 更新。
- 短内容在标题和播放器底栏之间尽量居中；长句和横屏允许页面纵向滚动，不裁切内容。

### 2.2 全文

- 只渲染现有 `.transcript`，不重复显示大型当前句卡片。
- 当前句继续使用 `.segment.active`，点击字幕继续调用 `seekToSegment`。
- 进入全文时将当前句滚动到播放器底栏以上的可见区域。
- 默认跟随播放：当前句离开安全可视区域时，将其滚动回接近视口中部。
- 用户通过触摸或滚轮主动纵向浏览后暂停跟随，并显示“回到当前句”；点击后定位当前句并恢复跟随。
- 每次重新进入全文时恢复跟随。

阅读模式继续复用 `showTranslation` / `show-translation`，在两个视图中保持一致。

## 3. 状态

在 `App.tsx` 中增加局部状态：

```ts
type PlayerView = "focus" | "transcript";

const [playerView, setPlayerView] = useState<PlayerView>("focus");
const [followingTranscript, setFollowingTranscript] = useState(true);
```

- 打开、切换或退出课程时重置为 `focus`。
- 进入全文时将 `followingTranscript` 重置为 `true`。
- 用户通过触摸或滚轮主动浏览全文后将其设为 `false`，不能用程序滚动也会触发的普通 `scroll` 事件判断。
- 两个状态都不持久化。
- 切换视图不得修改 `currentTime`、`playing`、`loopSegment`、`repeatIteration` 或循环切换锁。

当前字幕元素使用单个 `ref` 保存。

## 4. 全文定位与性能

当前高亮字幕复用单个 ref：

```ts
const activeSegmentRef = useRef<HTMLButtonElement>(null);
```

进入全文或跟随播放时，仅在当前句离开顶部内容区与固定底栏之间的安全区域后滚动。使用
`scrollIntoView({ block: "center" })` 时必须配合底栏留白或 `scroll-margin`，确保目标不被遮挡；
`prefers-reduced-motion: reduce` 下取消平滑滚动。

全文列表不能因 `currentTime` 的连续更新而每次重新执行完整 `map`。可直接在 `App.tsx` 使用 `useMemo`，只在
`course`、`currentSegment`、`showTranslation` 等真正影响列表的值变化时重新生成。不拆新组件，不引入虚拟
列表；如长课程仍有性能问题，再单独设计虚拟化。

## 5. 与播放器的关系

- Feature 0007 的底栏结构、菜单、安全区和时间轴保持不变。
- 核心播放按钮左侧提供设置栏开关；收起后隐藏倍速、逐句循环和阅读模式入口，并缩短底栏占用空间。
- 收起设置栏会关闭已打开的设置菜单，但不改变倍速、循环、阅读模式、播放状态或位置。
- 点击正文切换按钮属于菜单外部点击，按现有规则关闭播放器菜单。
- Feature 0006 的循环目标、遍数、统一完成逻辑和切换锁保持不变。
- 全文点击字幕继续沿用现有 seek 和循环重置语义。
- 前台 RAF、后台 `onTimeUpdate`、锁屏、下载、离线和进度保存不变。

## 8. 验收标准

- 默认只显示当前句，并且只能通过顶部按钮进入全文。
- 当前句与全文不会同时展示。
- 全文默认跟随当前句，用户浏览后暂停，并可恢复跟随。
- 设置栏可以展开或收起，收起不改变任何播放设置。
- 左右滑动不会切换正文视图。
- 视图切换不影响播放、seek、循环、下载、离线和进度。
- 长句和窄屏不裁切，全文最后一句不被底栏遮挡。
- 完整字幕不会随每次 `currentTime` 更新重新生成。
- 不增加依赖、路由、数据库字段或新的架构分层。

## 10. iOS 主屏幕 PWA 锁屏循环修复

### 10.1 问题

同一课程在 iPhone 浏览器标签页中锁屏后可以继续逐句循环，但通过主屏幕图标以 standalone 模式运行时，
锁屏后可能在当前句结束时漏掉下一遍循环。

原实现假设锁屏后 `document.hidden` 一定为 `true`：页面可见时由 `requestAnimationFrame` 检测句尾，只有
`document.hidden` 为 `true` 时才允许 `timeupdate` 检测句尾。iOS 主屏幕 Web App 的页面可见性状态在锁屏
场景中可能仍报告为可见；与此同时，动画帧已经被系统冻结，因此两条句尾检测路径会同时失效。

此外，原实现使用同一个可见性状态决定是否跳过 300ms 计时器。后台计时器可能被冻结，如果可见性状态
错误，播放器可能在句尾暂停后无法恢复。

首次修复后，后台路径虽能检测到句尾，但统一处理函数仍先调用 `audio.pause()`，再 seek 并调用
`audio.play()`。iOS 主屏幕 Web App 息屏后可能不允许脚本重新激活音频输出，表现为第一次结束后无声，
即使系统或页面显示的播放进度仍在变化。后台循环不能主动中断正在工作的媒体播放会话。

### 10.2 修复

- `timeupdate` 不再以 `document.hidden` 作为进入统一句尾处理函数的前置条件；
- 动画帧每次运行时记录时间；`timeupdate` 发现动画帧超过 500ms 未运行时，将页面视为实际已冻结；
- 页面隐藏或动画帧停止时，句尾处理立即 seek 并继续播放，不依赖后台计时器；
- 后台立即切换时不调用 `pause()`，只修改 `currentTime`，保持 iOS 音频播放会话连续；
- 仅前台 300ms 间隔和有限循环完成最后一句时主动暂停音频；
- 动画帧仍活跃时保留前台 300ms 循环间隔；
- 两条检测路径继续共用 `completeRepeatIteration` 和 `repeatSwitchingRef`，避免同一片段重复计数。

### 10.3 验证

1. iPhone 浏览器标签页前台和锁屏后，无限循环持续回到当前句。
2. iPhone 主屏幕 Web App 前台和锁屏后，无限循环持续回到当前句。
3. 有限循环在两种运行模式下都能按次数进入下一句，最后一句完成后暂停。
4. 前台循环仍保留约 300ms 间隔，后台切换不依赖 `setTimeout`。
5. RAF 与 `timeupdate` 同时越过句尾时，每一遍只计数一次。
6. 息屏跨越句尾后声音持续输出，不出现进度变化但无声的状态。
