# Feature 0011：界面视觉系统与图标统一

状态：已实现

## 1. 结论

在不改变信息架构、播放行为和存储结构的前提下，统一应用的图标、颜色、按钮状态、层级和基础质感。

本 Feature 采用“浅灰白背景 + 近黑正文 + 珊瑚红强调色”的现代浅色音乐播放器风格，将散落在 CSS 中的颜色整理为
语义变量，并用同一套 SVG 图标替换界面中的 Unicode 符号和不一致的字符图标。播放器主播放按钮继续使用实心图标，
以保留最强的操作层级。

目标不是把应用改造成另一种产品，也不引入 UI 框架；完成后界面应更克制、统一、清晰，并继续适合 iPhone 单手操作。
课程目录采用“浅色音乐播放器下的扁平分组列表”：参考成熟音乐应用的内容优先、无框列表和弱容器方法，但不复制其品牌、
标签、会员入口或复杂操作。

## 2. 当前问题

当前界面已经具备清楚的内容结构，但视觉细节还没有形成完整规则：

1. 播放器混用 `⌄`、`⌃`、`‹`、`›`、`▶` 和 `Ⅱ` 等字体字符；不同系统字体下的粗细、基线和尺寸不一致。
2. 课程分组使用 `+` / `−`，返回入口使用 `←`，菜单选中使用 `✓`，与播放器符号不是同一种视觉语言。
3. 旧界面的深绿、琥珀、多组灰色、边框色和白色透明背景以具体色值散落在 CSS 中，同一语义有多个接近但不一致的颜色。
4. 默认、悬停、按下、键盘聚焦、选中和禁用状态没有统一规则；部分按钮主要依靠背景颜色表达状态。
5. 课程目录使用“分组圆角框 → 内层列表留白 → 课程圆角卡片 → 下载描边按钮”的多层嵌套；边框、圆角和留白
   重复表达同一层级，使列表厚重、割裂，并降低窄屏的信息密度。
6. PWA 图标已有扬声器图形和现有配色，但网页内没有复用同一套图形语言。

## 3. 设计原则

- **内容优先**：课程标题和英文字幕仍是页面视觉中心，装饰不能与学习内容竞争。
- **颜色有语义**：近黑表达标题和英文正文；珊瑚红只表达播放、进度、选中和少量交互强调；灰色表达中文翻译与元数据；
  危险操作使用独立的深红语义。
- **图标一致**：普通操作统一使用圆角描边图标；播放、暂停等媒体核心状态可以使用实心图标。
- **状态不只靠颜色**：选中项同时使用图标、形状或文字状态；所有纯图标按钮保留可访问名称。
- **移动端优先**：触控目标尽量不小于 44 CSS 像素，图标的可见图形通常为 20–24px。
- **弱化容器**：列表和单句阅读区优先使用字号、间距和细分隔线建立层级，只在浮层等确有必要的区域使用卡片表面。
- **保持轻量**：不增加图标库、UI 组件库、主题框架或运行时 CSS 依赖。

## 4. 颜色系统

### 4.1 语义变量

在 `:root` 中建立语义变量，组件样式只引用变量，不再新增零散色值。最终实现时可以在浏览器中微调具体数值，
但语义和使用边界保持如下：

```css
:root {
  --color-canvas: #f7f8fa;
  --color-surface: #ffffff;
  --color-surface-muted: #f0f1f3;
  --color-text: #202124;
  --color-text-secondary: #6b7280;
  --color-text-disabled: #a6abb3;
  --color-accent: #d93b3b;
  --color-accent-hover: #c83232;
  --color-accent-soft: #fdecec;
  --color-border: #e5e7eb;
  --color-border-strong: #cfd3d8;
  --color-danger: #c93636;
  --color-danger-soft: #fdeaea;
  --color-danger-border: #e3a0a0;
  --color-on-accent: #ffffff;
}
```

页面使用干净的浅灰白底色，不保留旧的暖黄色径向渐变。播放器底栏使用带透明度和模糊的白色表面。

### 4.2 使用规则

- 页面背景、必要卡片、浮层和固定底栏分别使用 canvas、surface 与透明 surface，形成清楚但轻微的层级；课程列表
  不用 surface 卡片逐条包裹内容。
- 主操作、播放按钮、时间轴和选中状态使用 accent；不大面积填充页面背景。
- 英文、标题和关键数值使用 text；中文翻译、课程元数据和辅助说明统一使用 text-secondary。
- 禁用状态使用 text-disabled，但仍需能辨认。
- 删除按钮和错误消息使用 danger；普通离线、下载和播放状态不得借用危险色。
- 正文文本、按钮文字和必要状态满足 WCAG AA；键盘焦点环在相邻背景上必须清晰可见。

本 Feature 只实现一套浅色主题。深色模式需要单独验证字幕阅读、底栏与 iOS 状态栏，不在本次范围内。

## 5. 图标系统

### 5.1 实现方式

使用项目内的 React SVG 图标，不安装第三方图标包。图标使用统一的 `24 × 24` viewBox、圆角端点和圆角连接；
普通图标默认 `stroke="currentColor"`，实心媒体图标使用 `fill="currentColor"`。

图标数量足以形成重复模式，因此集中放在一个小型 `src/icons.tsx` 文件中。该文件只包含静态 SVG 组件，不包含按钮、
业务状态或样式逻辑。图标组件默认添加 `aria-hidden="true"` 和 `focusable="false"`，可访问名称由所属按钮提供。

不从外部 CDN 加载字体或 SVG，不使用 emoji，也不使用依赖当前字体字形的箭头和播放符号。

### 5.2 首批图标

| 场景 | 图标 | 说明 |
| --- | --- | --- |
| 返回课程 | `ArrowLeft` | 替换文字前的 `←`，保留“返回课程”文本 |
| 课程分组 | `ChevronDown` | 通过旋转表达展开状态，替换 `+` / `−` |
| 下载课程 | `Download` / `Check` | 文字继续显示“下载”或“已下载”，图标作为辅助状态 |
| 展开播放器设置 | `Sliders` / `SlidersExpand` | 使用两个 SVG 状态并保留 `aria-expanded`，不再使用 `⌄` / `⌃` |
| 上一句 / 下一句 | `SkipBack` / `SkipForward` | 语义是句子切换，不表现为五秒快退或快进 |
| 播放 / 暂停 | `Play` / `Pause` | 使用实心媒体图标，替换 `▶` / `Ⅱ` |
| 菜单选中 | `Check` | 与背景高亮共同表达选中状态 |
| 回到当前句 | `Locate` | 与文字一起显示，不做纯图标悬浮按钮 |
| 单句视图 | `SingleSentence` | 聚焦角包围一条句子线，表达一次只显示一句 |
| 全文视图 | `FullText` | 三条文本线，表达完整字幕内容 |
| 删除本地课程 | `Trash` | 继续显示危险操作文字，避免误触和语义不清 |

PWA `public/icon.svg` 保留扬声器主题，使用珊瑚红底和白色图形，并检查 iOS 主屏幕裁切；
本 Feature 不新增多套启动图、插画或课程封面。

## 6. 组件视觉调整

### 6.1 课程列表

课程目录采用内容优先的扁平分组列表。目标结构如下：

```text
[展开状态] 分组名称                              72 课

课程标题                                      [下载]
时长 · 大小
──────────────────────────────────────────────────
课程标题                                      [下载]
时长 · 大小

[收起状态] 下一个分组名称                      96 课
```

- 移除 `.course-group` 的外层背景、四周边框、大圆角和阴影；分组不再表现为卡片。
- 移除 `.course-group-list` 用来制造内层卡片区域的背景和多余包裹留白，课程行直接排列在分组标题下面。
- 移除每个 `.course-row` 的独立背景、四周边框、大圆角、悬浮阴影和上浮位移。
- 分组之间使用清楚的纵向留白；同组课程之间使用留白或贯穿内容区的极淡 1px 分隔线，不同时使用两者制造厚重层级。
- 分组标题保持可展开和收起，展开 SVG、分组名称与课程数量放在同一行。标题行自身提供不小于 44px 的点击区域，
  不依赖外框表达可点击性。
- 课程标题和时长/大小左对齐；下载状态位于右侧并纵向对齐。课程标题是主要信息，元数据与下载状态使用次级层级。
- 整个课程正文区域仍可点击打开课程；右侧下载入口必须阻止打开课程的操作，不改变现有下载语义。
- “下载”改为无外框的 Download SVG + 文字；“已下载”使用 Check SVG + 状态文字。两者保留至少 44px 点击区域，
  不能只通过颜色区别状态。
- hover 和 active 只显示轻微的整行底色，不增加边框或阴影；触屏下不依赖 hover。
- 键盘焦点使用 2px focus ring 和明确 offset。焦点可落在课程入口与下载入口，且不能因取消卡片边框而变得不可见。
- 320px 窄屏下课程标题可以自然换行，右侧下载入口保持可操作；不允许通过缩小字号或触控区域强行维持单行。

本次只借鉴成熟音乐应用的扁平列表结构，不增加封面、标签、收藏、评论、更多菜单、广告或会员入口。课程数据、分组顺序、
展开行为、在线播放和离线下载流程保持不变。

### 6.2 课程页头和正文视图

- 初始页头使用 44px 的 ArrowLeft 纯图标按钮，保留 `aria-label="返回首页"`，不重复显示返回文字。标题采用紧凑的无衬线样式。
- “单句 / 全文”使用轻量文字标签；当前项同时使用近黑文字和珊瑚红下划线，不再使用大面积实色分段按钮。
- 单句视图采用无框、无背景、无阴影的顶部对齐阅读区，不使用垂直居中。模式切换栏下方约 24px 作为稳定阅读起点，
  自动播放和手动切句时英文始终从同一顶部位置开始，避免用户反复寻找视线锚点。
- 单句元信息不重复显示“当前句”：使用珊瑚红的 `当前序号 / 总数`，可选说话人使用次级灰，并用圆点分隔。英文保持主要层级；
  中文位于英文下方约 16px，不使用水平分隔线。
- 切句只使用约 120ms 的轻微透明度变化，不做上下位移；长句仍由页面自然滚动，不建立独立滚动容器。
- 全文采用歌词式当前行强调，不给当前句增加卡片、边框或阴影。未播放英文使用次级灰；当前英文切换为近黑并略加粗，
  当前时间同时使用珊瑚红，因此状态不只依靠颜色表达。英文长句和双语内容继续左对齐。
- “回到当前句”使用 Locate + 文本，并继续避开固定播放器和安全区。

长全文滚动时增加方向感知的紧凑顶部栏：

- 页面继续向下阅读（`scrollY` 增加）时隐藏顶部栏，不占用字幕阅读空间。
- 用户反向滚动查看前文（`scrollY` 减少）累计约 12px 后显示顶部栏；向下滚动累计约 16px 后再次隐藏，过滤触控惯性和微小抖动。
- 只响应 wheel、touchmove 和相关键盘滚动等用户输入，不因字幕自动居中或点击跳句产生的程序化滚动而意外出现。
- 紧凑栏固定覆盖在页面顶部，不改变文档流，不推动正文，并保持单行约 52px 高度。内容依次为 44px 返回按钮、短分隔线、
  “单句 / 全文”两个中文文字按钮和占据剩余空间的单行省略标题，让用户无需回到页首即可改变阅读模式。
- 紧凑栏与页面顶部使用相同的“单句 / 全文”命名和文字标签样式，避免用户再次学习抽象图标；当前模式使用近黑文字、
  珊瑚红下划线和 `aria-pressed`，不使用浅红块状背景。按钮分别使用 `aria-label="单句视图"` 与 `aria-label="全文视图"`。
- 使用半透明白色、轻微模糊和底部分隔线，并适配 `env(safe-area-inset-top)`；显示和隐藏过渡为 150–180ms。
- 页面接近顶部时隐藏紧凑栏，使用文档中的正常页头，避免两个返回入口同时出现。

### 6.3 播放器底栏

- 主控制区仍为“设置、上一句、播放/暂停、下一句、占位”五列，不改变 Feature 0007 和 0008 的结构。
- 播放按钮继续是最大、唯一的珊瑚红圆形按钮；播放和暂停图标使用相同视觉尺寸和光学居中。
- 其他主控制按钮使用浅灰表面、近黑图标；hover、active 和 focus-visible 状态统一。
- 设置入口使用图标旋转或状态变化表达展开/收起，并保留 `aria-expanded` 和动态 `aria-label`。
- 时间轴继续使用原生 `input[type="range"]`，统一 accent、轨道与拇指颜色；不为了自定义外观重写拖动行为。
- 倍速、逐句循环和阅读模式仍保留文字值与标签。它们是信息型设置，不改成难以理解的纯图标。
- 弹出菜单的选中项统一使用 Check + 背景高亮，菜单边框、圆角、阴影与其他浮层使用同一层级变量。

## 7. 交互与可访问性

- 纯图标按钮必须有准确的 `aria-label`；图标自身不进入可访问性树。
- 带文字按钮不重复朗读图标名称。
- `aria-expanded`、`aria-pressed`、`aria-checked` 和现有 menu role 保持有效。
- 键盘焦点不能只依赖浏览器不稳定的默认轮廓；使用全局统一的 `:focus-visible` 样式。
- hover 只作为补充，所有状态在触屏和键盘下都可识别。
- 动画限制为 120–180ms 的颜色、位移或旋转过渡；`prefers-reduced-motion: reduce` 下关闭非必要动画。
- 不用图标替代“删除本地课程”等高风险操作的文字，不增加长按、滑动或隐藏手势。
- 隐藏的紧凑顶部栏不得接收指针或键盘焦点；显示后返回按钮和阅读模式按钮仍具备准确状态、可访问名称与清晰的
  `focus-visible` 状态。

实现内容：

2. 将现有色值迁移为语义 CSS 变量，统一表面、边框、文字、强调、危险和焦点颜色。
3. 统一按钮、图标按钮、菜单选中、键盘焦点和禁用状态。

5. 让 PWA 图标与网页内颜色保持一致，并核对 manifest 的主题色与背景色。

不修改播放状态机、字幕定位、课程 Schema、IndexedDB、OPFS、Service Worker 缓存边界或偏好存储。

10. 点击课程正文仍打开课程，点击下载只执行下载；键盘可分别聚焦并操作两个入口。

    并且字幕自动跟随不会触发顶部栏。
13. 紧凑顶部栏适配 iPhone 安全区，长标题单行省略，不遮挡正文交互；减少动态效果时不执行位移动画。

## 10. 验收标准

- 界面中不再使用 Unicode 字符充当返回、展开、切句、播放、暂停或菜单选中图标。
- 所有操作图标来自同一套本地 SVG 规则，粗细、尺寸和对齐一致。
- CSS 颜色以语义变量为唯一来源，组件不再散落新增品牌和状态色值。
- 主播放操作、普通操作、选中状态、危险操作和禁用状态具有清晰且一致的层级。
- 课程目录是无外层卡片、无逐条卡片的扁平分组列表，不再出现“容器套卡片套按钮边框”的嵌套结构。
- 分组标题、课程标题、元数据和下载状态仅通过字号、间距、颜色与单一分隔线形成清晰层级。
- 课程列表、当前句、全文、菜单和底栏在视觉上属于同一套系统，但不要求所有区域都使用卡片。
- 全文当前句与其他字幕处于同一平面，不出现独立方框；当前英文通过近黑、字重和红色时间共同突出。
- 长全文反向滚动时可从紧凑顶部栏直接返回首页，无需先滚动到文档顶部；继续向下阅读时顶部栏自动收起。
