# Feature 0014：按字母权重估算的整词播放进度

## 状态

已实现。

## 背景

现有字幕只提供句子级开始和结束时间。播放器可以准确定位当前句，但长句播放时，用户仍难以判断声音大致对应到哪个英文单词。

真正的逐词时间轴需要对所有音频执行强制对齐并扩展课程数据。本 Feature 先提供一个不修改课程 JSON 的轻量版本：根据当前句内播放比例和每个单词的字母数量估算当前位置，并始终以完整单词为单位显示高亮。

## 需求分析

### 用户问题

- 长句在屏幕上占据多行，只有句子级高亮时定位范围过大；
- 用户听到连读、弱读或生词后，很难快速把声音对应到文字；
- 手动拖动进度时，缺少比“当前句”更细的视觉反馈。

### 方案价值

- 所有现有材料立即可用，不需要重新制作字幕；
- 新概念等语速稳定的朗读材料可以获得较好的近似效果；
- 即使存在一至数词误差，也能显著缩小用户查找范围；
- 后续接入真实逐词时间戳时，可以复用同一套文字视觉组件。

### 准确性边界

该方案不是语音识别或真实歌词时间轴。停顿、重读、口误、连读和字幕改写都会造成偏差，播客中的误差可能达到一至三个词。

界面必须使用克制的辅助高亮，不能通过逐字填充等方式制造“精确同步”的错觉。没有真实时间数据时，产品不显示“精准逐词同步”等文案。

## 目标

1. 根据当前播放时间估算当前句内对应的英文单词。
2. 时间分配按单词包含的字母和数字数量加权，而不是每个词平均分配。
3. 当前单词始终完整高亮，不进行字母级或单词内部渐变。
4. 单句和全文模式都能看到相同的当前词反馈。
5. 手动切句、点击字幕和拖动进度后立即更新。
6. 长播客播放时不得因逐词进度而反复重新渲染全部字幕。

## 非目标

- 不修改课程 JSON Schema；
- 不生成或保存逐词时间戳；
- 不实现音节、音素或字母级进度；
- 不让用户点击单词跳转；
- 不同步中文译文；
- 不提供逐词高亮开关；
- 不替代未来的真实强制对齐方案。

## 分词规则

英文句子先拆为“单词”和“非单词文本”两类 token，渲染时保持原始顺序和空格。

### 单词

- 连续 Unicode 字母或数字构成一个单词；
- 单词内部的英文撇号、弯撇号和连字符允许连接两侧字符；
- `don't`、`it's`、`state-of-the-art` 均作为一个完整单词；
- 计算权重时只计算字母和数字，撇号和连字符不计入。

### 非单词文本

- 空格、逗号、句号、问号、引号和其他标点原样保留；
- 非单词文本不参与权重计算；
- 标点不单独获得高亮状态。

### 最小权重

每个单词的权重至少为 `1`，避免异常 token 得到零时长。

## 估算算法

对当前句：

```text
句内比例 = clamp((当前时间 - 句子开始) / (句子结束 - 句子开始), 0, 1)
总权重 = 所有单词的字母和数字数量之和
目标权重 = 句内比例 × 总权重
```

从第一个单词开始累加权重，第一个累计值大于目标权重的单词就是当前词。

边界规则：

- 当前时间小于或等于句子开始时，当前词为第一个单词；
- 当前时间到达句子结束时，最后一个单词进入已播放状态，不再保留当前词；
- 没有可识别单词时，整句按普通文本渲染；
- 句子时长无效时，回退到第一个单词；
- 当前时间始终限制在句子时间范围内。

## 视觉设计

### 当前词

- 使用现有强调红色；
- 使用非常浅的强调色背景辅助识别，不使用下划线；
- 圆角和内边距不得造成明显的行高变化；
- 不改变字号和字体粗细，避免句子在播放时发生排版抖动。

### 已播放词

- 使用正常正文颜色。

### 未播放词

- 使用正文次级色，但保持足够对比度；
- 不降低到禁用态颜色，避免整句难以阅读。

### 标点和空格

保持正文颜色，不随播放状态闪烁。

## 单句模式

- 当前句英文按 token 渲染；
- 播放时整词高亮随估算位置移动；
- 中文翻译保持现状；
- 切换到新句后从该句第一个单词开始。

## 全文模式

- 只有当前句的英文拆成单词 span；
- 其他句子继续作为普通文本渲染；
- 当前句改变时，上一句恢复普通文本，新句开始逐词显示；
- 单词推进时不得重新创建全部字幕列表。

## 更新机制与性能

### 时间来源

前台播放时以共享 `HTMLAudioElement.currentTime` 为真实时间来源，不依赖频率较低的 React `currentTime` 状态。

### 动画帧

- 播放期间使用 `requestAnimationFrame` 检查估算词；
- 只有当前词序号变化时才更新 DOM class；
- 暂停、组件卸载或课程切换时取消动画帧；
- 浏览器后台暂停动画帧后，回到前台按真实音频时间直接追上。

### 避免全文重渲染

当前词序号不进入会导致整个字幕数组重建的 React state。单句正文和全文当前句分别持有容器 ref，动画帧只切换其中 `.timed-word` 的状态 class。

句子切换仍使用现有 React 状态和二分查找，保持播放器现有数据流。

## 播放操作

- 点击某句：切换句子后立即定位该句第一个词；
- 上一句/下一句：立即刷新词进度；
- 拖动进度条：按新的句内比例重新估算；
- 逐句循环回到句首：词进度回到第一个词；
- 倍速只影响实际时间推进速度，不改变估算公式；
- 暂停时保持当前词，不运行持续动画。

## 无障碍

- 单词 span 不增加按钮或链接语义；
- 屏幕阅读器仍按完整英文句子朗读，不逐词播报状态变化；
- 不使用 `aria-live` 宣告每次词变化；
- 高亮同时使用颜色与背景/下划线，不只依赖颜色；
- `prefers-reduced-motion` 不需要关闭词状态更新，因为没有位移动画。

## 兼容与降级

- 所有现有课程无需迁移；
- 不包含英文单词的句子保持普通文本；
- 不支持 Unicode 属性正则的目标浏览器不在当前支持范围；现代 iPhone Safari 和桌面浏览器均支持；
- 未来课程提供真实 `words` 时间戳时，应优先使用真实数据，并保留本算法作为缺失数据的回退。

## 实现建议

1. 增加纯函数 `tokenizeTimedText()`，保留原始文本并计算单词权重。
2. 增加纯函数 `findEstimatedWordIndex()`，根据句内时间返回单词序号。
3. 增加渲染辅助函数，为完整单词生成 `.timed-word` span。
4. 在播放器中增加单句和全文当前句两个 ref。
5. 使用独立动画帧同步 class，并在现有 seek 路径中安排一次即时刷新。
6. 不新增依赖，不修改数据库和课程文件。

## 验收标准

1. 播放长句时，当前完整单词按字母权重依次高亮。
2. `don't` 和带连字符的词不会被拆成多个高亮单位。
3. 标点、引号和空格保持原始显示。
4. 单词内部不会出现部分填色。
5. 已播放、当前和未播放词有可辨识但克制的视觉差异。
6. 单句和全文模式显示相同的估算位置。
7. 点击字幕、切换前后句和拖动进度后，高亮立即定位。
8. 逐句循环回到句首时，高亮回到第一个词。
9. 全文模式下，只有当前句创建单词 span。
10. 长播客播放时，单词推进不会重新构建完整字幕数组。
11. 暂停和切换材料后不存在遗留动画帧。
12. 320px 窄屏下英文换行稳定，无横向溢出。
13. `pnpm build` 与 `git diff --check` 通过。
