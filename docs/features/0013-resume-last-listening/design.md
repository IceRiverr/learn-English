# Feature 0013：继续收听与上次播放恢复

## 状态

已实现。

## 背景

Feature 0012 已经让播放会话在“播放页 → 听力库”的页面切换中持续存在，用户返回听力库后仍能通过底部迷你播放器控制当前材料。

但这个会话目前只存在于单次应用运行期间。刷新页面、关闭浏览器或重新打开主屏幕 Web App 后，应用虽然仍保存了每篇材料的播放时间和倍速，却不知道最后播放的是哪一篇，因此：

- 首页不会恢复迷你播放器；
- 用户需要重新找到材料并进入详情页；
- 已保存的进度只有在重新打开同一篇材料后才会生效；
- 产品体验仍像“课程目录”，没有形成音乐播放器常见的“随时继续”闭环。

## 需求分析

### 用户问题

用户进行精听时，一篇材料可能需要多次完成。特别是长篇播客，单次学习很难听完。再次打开应用时，如果必须重新展开分组、查找标题并进入材料，会产生不必要的操作成本，也容易让用户忘记上次听到哪里。

用户真正需要恢复的不是“上次打开过的页面”，而是“上次有效的播放上下文”：

1. 上次播放的是哪一篇材料；
2. 播放到了什么时间和哪一句；
3. 当时使用的倍速；
4. 当前材料是否仍可在线或离线播放。

### 产品价值

该功能属于高频、低认知成本的连续性功能：

- 缩短再次进入学习状态的路径；
- 对长材料和播客的价值尤其明显；
- 与现有自动保存进度、首页迷你播放器自然衔接；
- 让应用更接近用户熟悉的音乐和播客播放器，而不是每次从目录重新开始。

### 原始方案评审

原始设想包括两部分：

1. 在听力库中增加上次播放的材料；
2. 下次访问时，让迷你播放器定位到上次进度。

两部分都应保留，但承担不同职责：

- “继续收听”是可发现的内容入口，让用户看见上次材料、时间和恢复状态；
- 迷你播放器是操作入口，让用户直接播放、进入详情页或跳到下一句。

不建议只恢复迷你播放器。单靠底栏不容易解释为什么它会出现，也无法完整表达离线不可用、材料已完成等状态。

### 首页位置

“继续收听”应放在首页标题说明之后、完整听力列表之前，而不是放在“其他”分组底部。

原因：

- 它是跨分组的个人状态，不属于“其他”内容分类；
- 它的主要价值是减少查找，因此不能要求用户先滚动到页面底部；
- 首页顶部符合音乐、播客和有声内容产品对“继续播放”的常见优先级；
- 只显示一条记录，不会明显挤压内容列表。

## 目标

1. 成功打开一篇材料后，记录它为最后播放材料。
2. 刷新、关闭或重新打开应用时，在首页恢复“继续收听”。
3. 在条件允许时，恢复底部迷你播放器，并定位到已保存时间和当前句。
4. 恢复过程绝不自动播放音频，必须等待用户操作。
5. 对未下载材料的离线情况、材料被移除和已播放完成等状态提供明确降级。

## 非目标

- 不实现多条播放历史或最近播放列表；
- 不同步到账号或其他设备；
- 不持久化正在进行的逐句循环会话；
- 不恢复应用关闭前正在显示的页面，启动后仍进入听力库；
- 不在后台自动下载上次材料；
- 不增加路由、全局状态管理或新的状态依赖。

## 术语与文案

- 模块标题使用“继续收听”，不使用“上次播放的课程”。
- 应用继续使用“材料”或“一篇”，避免重新强化上课感。
- 普通未完成状态的主操作文案为“继续播放”。
- 已完成状态的主操作文案为“重新播放”。
- 在线材料离线不可用时显示“连接网络后继续”。

## 信息架构与界面

### 继续收听卡片

卡片位于首页标题下方、听力内容分组上方，显示：

- 材料标题；
- 当前句的英文摘要；能获得翻译时不在卡片重复展示中文；
- `当前时间 / 总时长`，例如 `10:23 / 154:56`；
- 一条细进度线；
- “继续播放”或“重新播放”状态。

整块卡片可进入完整播放页。卡片不单独放置倍速、循环和翻译控制，避免与迷你播放器重复。

### 迷你播放器

恢复成功后，继续使用 Feature 0012 的底部迷你播放器，不创建第二套播放器：

- 进度线定位到保存位置；
- 标题和当前句与“继续收听”保持一致；
- 播放按钮初始显示为播放状态；
- 点击播放从保存位置开始；
- 点击标题区域进入完整播放页；
- 点击下一句后，从下一句开始播放。

### 加载状态

应用启动时不应因为恢复流程阻塞整个听力库：

1. 首页和课程分组先正常渲染；
2. 后台读取最后播放记录和进度；
3. 恢复期间“继续收听”区域可使用轻量骨架或暂不显示；
4. 恢复成功后再显示卡片和迷你播放器；
5. 恢复失败不能影响用户打开其他材料。

不使用全屏加载状态，也不让所有课程按钮进入禁用状态。

## 数据设计

### 复用现有数据

继续复用：

- `course:<id>`：已下载课程及字幕；
- `audio:<id>` 或 OPFS：已下载音频；
- `progress:<id>`：`currentTime` 与 `playbackRate`。

不在最后播放记录中复制播放时间，避免两份进度产生不一致。

### 新增最后播放索引

在现有 IndexedDB `app-data` store 中新增固定键：

```text
last-played
```

值结构：

```ts
interface LastPlayed {
  courseId: string;
  updatedAt: number;
}
```

规则：

- 仅在材料成功加载并建立播放会话后写入；
- 播放、暂停和时间更新不重复写该索引；
- `updatedAt` 为将来扩展最近播放保留信息，本 Feature 只使用一条记录；
- 读取后必须检查字段类型，非法数据按无记录处理；
- 删除当前下载资源时，如果它仍是内置在线材料，保留最后播放索引和学习进度；如果材料已从内置目录移除且本地资源也被删除，则同时清除索引和进度。

### 写入时机

调用现有材料打开流程并成功完成以下步骤后写入：

1. 字幕已经通过 Zod 校验；
2. 已确定可用的本地 Object URL 或在线音频 URL；
3. `course` 和 `audioUrl` 可以建立播放会话。

下载材料但没有打开，不应改变“继续收听”。

## 冷启动恢复流程

应用启动后按以下顺序恢复：

1. 读取 `last-played`。
2. 使用 `courseId` 在内置 `lessons` 中查找材料。
3. 读取 `progress:<id>`；无进度时使用 `0` 秒和 `1×`。
4. 优先尝试读取本地 `course:<id>` 与对应音频。
5. 本地资源完整时，创建 Object URL 并恢复播放会话。
6. 没有完整本地资源时，在联网状态下获取并校验字幕，音频继续使用内置在线 URL。
7. 将 `appView` 保持为 `library`，设置 `playing = false`。
8. `loadedmetadata` 后把 `currentTime` 设置为保存时间，并使用现有二分查找定位当前句。
9. 显示“继续收听”和迷你播放器，等待用户操作。

恢复过程必须复用现有材料加载逻辑，避免“用户点击打开”和“冷启动恢复”形成两套校验与资源选择规则。

## 自动播放策略

恢复后禁止自动调用 `audio.play()`：

- 避免用户打开应用后突然出声；
- 符合 iPhone Safari 和现代浏览器的用户手势限制；
- “上次正在播放”不等于“本次允许自动播放”。

应用只恢复位置和上下文。第一次播放必须来自用户点击“继续播放”、迷你播放器播放按钮或进入详情后的播放按钮。

## 进度与完成状态

### 未完成

保存位置小于材料时长的 98% 时，从该位置继续。

### 基本完成

保存位置达到材料时长的 98%，或距离结束不足 10 秒时，视为基本完成：

- “继续收听”显示“重新播放”；
- 迷你播放器定位到 `0`；
- 当前句定位到第一句；
- 不立即覆盖原进度；用户实际点击重新播放后再保存新的 `0` 附近进度。

采用“98% 或最后 10 秒”是为了兼容几分钟的短材料和数小时的长播客。

### 越界数据

若保存时间小于 `0`、不是有限数值或大于课程时长：

- 将位置限制在 `0..duration`；
- 无法判断时回到 `0`；
- 不向用户显示技术错误。

## 在线、离线与异常处理

### 已下载材料

课程、字幕和音频都存在时，应在离线状态下完整恢复卡片、当前句和迷你播放器。

### 未下载的在线材料

- 联网时正常获取字幕并恢复；
- 离线时显示仅包含标题、保存时间和“连接网络后继续”的继续收听卡片；
- 未得到可播放资源前不挂载迷你播放器，避免启动时产生浏览器音频错误；
- 用户恢复联网并点击卡片时，重新执行正常打开流程。

### 材料已从目录移除

- 如果仍有完整本地下载，允许继续播放，并标记为本地材料；
- 如果没有完整本地资源，清除 `last-played`，不显示失效入口；
- 该失败不能阻塞听力库或弹出持续错误。

### 资源损坏

本地课程存在但音频缺失时，视为不完整下载：

- 如果内置目录仍有该材料且联网，回退在线资源；
- 否则显示不可用状态；
- 不自动删除用户数据，除非用户明确执行删除下载。

### 删除下载

当前界面的操作名称是“删除下载”，其语义应是释放本地音频和字幕占用，而不是删除学习记录：

- 对仍存在于内置目录的材料，只删除 `course:<id>`、本地音频 Blob 或 OPFS 文件；
- 保留 `progress:<id>` 和 `last-played`，下次联网时仍可从原位置继续；
- 对已经不在内置目录中的本地材料，删除下载后将无法重新获取，因此同时清除对应进度和最后播放索引；
- 实现时应拆分现有“删除完整课程数据”的数据库操作，避免继续由一个函数混合两种语义。

## 与现有播放器规则的关系

- 本 Feature 延续 Feature 0012 的单一共享 `HTMLAudioElement`，不得额外创建用于恢复的音频元素。
- 恢复成功后，页面导航仍不得卸载该音频元素。
- 恢复时 `loopSegment` 为 `undefined`、`repeatIteration` 为 `1`，不延续上次逐句循环会话。
- 循环次数偏好和翻译显示偏好继续按现有规则恢复。
- 播放倍速来自 `progress:<id>`。
- 当前字幕继续使用现有二分查找定位。
- 替换本地 Object URL 时继续及时释放旧 URL。

## 无障碍与移动端

- “继续收听”卡片使用一个明确的主按钮语义，避免嵌套按钮。
- 可访问名称包含材料标题和“继续播放”或“重新播放”状态。
- 进度信息同时提供文本，不只依赖颜色和进度线。
- 所有操作目标不小于 44 CSS 像素。
- 继续收听卡片和课程列表必须为底部迷你播放器及 `safe-area-inset-bottom` 保留空间。
- 恢复完成后不自动移动键盘焦点，也不使用打断式实时播报。

## 隐私与性能

- 所有最后播放信息只保存在当前浏览器本地，不上传服务器。
- 只保存一个课程 ID 和时间戳，不形成详细收听历史。
- 启动恢复最多读取一篇材料，不扫描全部进度记录。
- 在线字幕请求应复用浏览器 HTTP 缓存；不因为恢复而下载完整 MP3。
- `preload="metadata"` 保持不变，禁止把完整音频读入 `AudioBuffer`。

## 实现建议

1. 在 `src/db.ts` 增加 `saveLastPlayed()`、`loadLastPlayed()` 和 `clearLastPlayed()`。
2. 将 `openLesson()` 中“选择本地或在线资源并建立会话”的逻辑提取为可复用的小函数。
3. 在应用初始化 Effect 中启动非阻塞恢复，并使用取消标记防止卸载后的状态更新。
4. 为“资源已识别但暂不可播放”保留轻量的继续收听展示状态，避免伪造完整 `Course`。
5. 继续使用现有 React 本地状态，不引入路由或全局状态库。

## 验收标准

1. 打开材料并播放到中间位置后关闭页面，再次访问时首页出现“继续收听”。
2. 恢复后的迷你播放器时间、进度线和当前句与保存位置一致。
3. 再次访问时音频保持暂停，不会自动出声。
4. 点击继续播放后从保存位置开始，而不是从 `0` 或上一句开始。
5. 点击继续收听卡片或迷你播放器标题后进入完整播放页，进度不重置。
6. 已下载材料在离线状态下能够完整恢复。
7. 未下载材料离线时显示“连接网络后继续”，且不产生循环音频错误。
8. 上次材料被移除且本地无资源时，不显示失效入口，其他材料仍可正常打开。
9. 基本完成的材料显示“重新播放”，用户点击后从开头开始。
10. 恢复不会延续逐句循环会话，但会保留倍速和翻译偏好。
11. 320px 宽度和 iPhone 安全区下，卡片、课程列表与迷你播放器不重叠。
12. `pnpm build` 与 `git diff --check` 通过。

## 后续可能扩展

以下能力不属于本 Feature，但新增的 `updatedAt` 可为未来设计提供基础：

- 最近播放列表；
- 收藏或稍后听；
- 多设备进度同步；
- 首页根据学习频率排序。

这些扩展必须另立 Feature，不在本次实现中提前增加数据结构或界面。
