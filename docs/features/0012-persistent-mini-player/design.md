# Feature 0012：跨页面持续播放与首页迷你播放器

## 状态

已实现。

## 目标

用户从材料播放页返回听力库时，当前音频继续播放，不因为页面切换而中断。听力库底部保留一个精简播放器，提供当前材料信息和最常用的播放控制，并允许用户回到完整播放页。

## 设计结论

- 将“当前页面”与“播放会话”分离。返回听力库只改变页面状态，不卸载 `HTMLAudioElement`，也不清空当前课程、播放进度和逐句循环状态。
- 完整播放页继续使用现有的底部播放器，保留倍速、逐句循环、阅读模式、进度和前后句控制。
- 听力库使用单行迷你播放器，显示当前材料标题、当前句摘要、播放/暂停和下一句。
- 点击标题与当前句区域，回到该材料的完整播放页；播放位置和播放状态保持不变。
- 切换到另一篇材料时，才保存并结束旧播放会话，加载新材料。
- 删除当前已下载材料时，必须先停止并清空当前播放会话，再删除本地资源。

## 交互与版式

### 返回听力库

- 播放页的返回箭头只执行页面导航。
- 正在播放时保持播放；暂停时保持暂停。
- 关闭临时弹出菜单，避免菜单状态被带回听力库。

### 迷你播放器

- 固定在听力库底部，并适配 `safe-area-inset-bottom`。
- 顶部使用 2px 进度线反馈播放位置。
- 左侧文本区域显示材料标题和当前句摘要，整块可点击并返回完整播放页。
- 右侧提供不小于 44 CSS 像素的播放/暂停与下一句按钮。
- 迷你播放器不放置倍速、循环、翻译等低频设置，避免底部区域再次变成完整控制面板。

## 状态与生命周期

- `appView` 只表示当前显示“听力库”还是“播放器”。
- `course`、`audioUrl`、`currentTime`、`currentSegment`、`playing` 和循环状态共同构成当前播放会话。
- 只要播放会话存在，共享的 `audio` 元素就始终挂载在应用根级 JSX 中。
- 页面导航不得释放 Object URL；切换材料、删除材料或组件卸载时才释放。
- 进度仍按现有节奏写入 IndexedDB，进入后台时继续保存。

## 无障碍

- 迷你播放器使用 `aria-label="迷你播放器"`。
- 标题区域提供包含当前材料名称的可访问名称。
- 播放状态通过播放/暂停按钮的 `aria-label` 表达。
- 所有控制均保持键盘可操作，触控目标不小于 44 CSS 像素。

## 验收标准

1. 播放音频后返回听力库，音频不暂停且播放时间继续增长。
2. 听力库底部出现迷你播放器，并显示当前材料和当前句。
3. 迷你播放器可播放/暂停、跳到下一句，并能返回完整播放页。
4. 从迷你播放器返回完整播放页后，播放时间不会归零，也不会重新加载当前材料。
5. 从听力库打开另一篇材料时，旧进度被保存，新材料正常加载。
6. 删除当前本地材料后，播放器被清空，底部迷你播放器消失。
7. 窄屏、iPhone 安全区和键盘操作均正常。
